**Description:** Track your browser extensions and competitors in one place

---

### 1. Who are the users?
**Everyday browser users who want to enhance their web experience** (B2C — individual users making a personal purchase decision).

### 2. Why do they need it?
They repeat tedious browser actions daily that a one-click extension could eliminate

  > What the product says: _"Track your browser extensions and competitors in one place"_

### 3. How does it find users?
**Chrome Web Store** organic search is the primary channel — users search for a task and your extension appears. Submit to extension directories (extlib.io, alternativeto). Zero ongoing CAC once listed.

### 4. Does it make money? How much?
Monetisation model not explicit from public info. For indie viability, target freemium with a **$9–$29/month** paid tier. Validate willingness to pay before building the billing system.

### 5. What can I learn from this?
- The Chrome Web Store is a search engine most developers forget to compete in — a well-optimised listing gets passive, recurring installs with zero ad spend.
- Extensions have structurally better retention than web apps: uninstalling requires deliberate effort, so users stick around.

### 6. One-sentence pitch

### 7. Can I build it?
**Yes** — Chrome extensions are HTML/CSS/JS. Any web developer can ship an MVP in a weekend. Google's Manifest V3 docs are excellent.
